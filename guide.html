GTS NR1 Utils - User's Guide

A Chrome extension for intercepting and inspecting NerdGraph and NRQL requests on New Relic pages.

Why Use This Extension?

When troubleshooting issues on the New Relic platform, technical support engineers typically need to piece together information from multiple sources — Chrome DevTools Network tab, browser console, NR1 debug panel, NerdGraph API Explorer, and the page UI itself. This extension consolidates that workflow into a single side panel.

What It Reduces the Need For

  • Searching through raw network payloads — The extension automatically captures, structures, and makes searchable all GraphQL and NRQL requests with their query names, variables, full responses, timing, and errors. No manual filtering, payload reading, or request/response correlation needed.
  • NR1 Debug Info panel — Platform version, nerdlet ID, nerdpack details, and team ownership are surfaced in the Debug Mode tab without needing to navigate to the NR1 debug panel or inspect page internals.
  • Manual HAR exports — Select specific results and export them as structured JSON with a Save As dialog. No need to export a full HAR and parse through hundreds of unrelated network requests.

Key Benefits for Technical Support

  • See errors immediately — Failed requests surface with red status indicators and error banners at the top of the detail view. No scrolling through JSON to find error messages.
  • Identify owning teams — When an owningTeam field is found in the JSON response, the extension automatically extracts and displays it, so you know who to escalate to without digging through the response payload. This banner will not appear if the response does not contain owning team information.
  • Spot slow queries in real time — Pending requests appear with a live timer the moment they fire. You can see which queries are taking longest before they even complete.
  • Detect cross-account queries — The extension flags when results are querying multiple account IDs, which can indicate misconfiguration or unexpected cross-account behavior.
  • Capture everything on page load — The extension intercepts requests from the moment the page starts loading, before you could even open DevTools. Nothing is missed.
  • Share findings easily — Export selected results as clean JSON files to attach to support tickets or share with engineering teams. The JSON includes the full query, variables, response, errors, and timing — everything needed to reproduce and investigate.
  • No context switching — The side panel stays open alongside the New Relic page. Click around in the UI and watch the requests flow in real time without switching between tabs or tools.

Getting Started

Click the extension icon in Chrome's toolbar to open the side panel. The extension automatically intercepts GraphQL and NRQL network requests on any *.newrelic.com page.

First time setup: After installing or re-enabling the extension, refresh any already-open New Relic tabs for the extension to begin capturing requests. New tabs opened after installation will work automatically.

The extension has three tabs:

  • NerdGraph requests — Captures all GraphQL queries and mutations
  • NRQL requests — Captures NRQL queries (including those embedded in GraphQL)
  • Debug Mode — Displays platform info, nerdpack metadata, and nerdlet details

Request List (Left Panel)

Live Request Status

Requests appear immediately when fired, with a live status indicator. Each request shows a query pill and a timing pill:

  • QUERY PENDING — Request in flight. Timer counts up in real time with a pulsing animation.
  • QUERY RESPONSE TIME — Request completed successfully with no errors.
  • QUERY RESPONSE TIME — Response received but contained errors. Timing stays green because a response was returned.
  • QUERY RESPONSE TIME — Request timed out. Both pills are red.

Sorting

Results are automatically sorted by status priority, then by most recent start time within each group:

  1. Pending (yellow) — Always at the top while waiting to resolve
  2. Errors/Timeouts (red) — Resolved requests with errors
  3. Success (green) — Resolved requests without errors

Search

The "Search for results" field at the top of the left panel filters results by searching across the entire request — query text, variables, response data, errors, and all other fields.

Multi-Account Detection

When results are found that query more than one account ID, a yellow notification banner appears listing the account IDs detected and the number of affected results. A toggle button lets you filter the list to show only those multi-account results.

Selecting Results

Each result has a checkbox for selection. Use these for exporting specific results:

  • Individual checkboxes — Select specific results
  • Select all — Toggles all visible (filtered) results. Shows an indeterminate state when some are selected.

When no results are selected, a hint message appears: "Select one or more results to export."

Result Detail (Right Panel)

Click any result in the list to view its full details in the right panel.

Banners

When applicable, information banners appear at the top of the detail view in the following order:

  • Owning Team (blue left border) — Shows the team responsible for the query, extracted from the owningTeam field in the JSON response. This banner only appears when the field is present in the response — not all responses include owning team information.
  • Account IDs Being Queried (green left border) — Lists the account IDs found in the response's account_ids arrays.
  • Error (red left border) — Displays error messages from the response, making them immediately visible without scrolling.

Raw JSON

The full request object is displayed as formatted JSON, exactly as it would appear in an exported file. This includes the query, variables, response, errors, timing, and all metadata.

Search Within Result

Use the "Search within result" field to find text within the JSON:

  • Matches are highlighted in yellow
  • The current match is highlighted in orange
  • Navigate with Enter (next) and Shift+Enter (previous)
  • Press Escape to close the search

Copy JSON to Clipboard

The COPY JSON TO CLIPBOARD button copies the entire result as formatted JSON. The button briefly shows "Copied!" to confirm.

Exporting Results

The EXPORT button exports selected results as a JSON file.

  • Select one or more results using the checkboxes
  • Click Export — a Save As dialog opens where you choose the file location and name
  • The default filename includes the request type and timestamp (e.g., nerdgraph-requests-2026-04-08T22-15-00.json)

When no results are selected, the Export button is greyed out with a tooltip: "Select at least one result to export."

Filters

  • Errors only — Show only requests that returned errors (any request with a non-null errors field)
  • Timeouts only — Show only requests that timed out (detected by searching error messages for the word "timeout")

These filters can be combined with the search field and the multi-account filter.

Clear Log

The CLEAR LOG button removes all captured results from the current tab. This is the only way to clear results — they persist across page navigation within the same tab.

Debug Mode

The Debug Mode tab displays information about the current New Relic page:

  • Platform Info — Platform version, environment, region, user ID, and account ID
  • Debug Info — Current nerdlet, nerdpack details, and SDK versions
  • Associated Teams — Team ownership information for the current nerdpack

Debug info refreshes automatically when navigating between NR1 pages.

Note: Debug info may take a few seconds to load after navigating to a new page. If it doesn't appear, try refreshing the page.

For a deeper look into the behavior of these features, see the Under The Hood button at the top left of this guide.

Known Limitations

  • IntrospectionQuery requests to nerd-graph.service.newrelic.com (used by the NerdGraph API Explorer) cannot be captured due to NR's browser agent intercepting the fetch promise chain.
  • Debug Mode info may not update during SPA (single-page app) navigation within NR1. Refreshing the page resolves this.

GTS NR1 Utils is a development tool built for the GTS team at New Relic.
